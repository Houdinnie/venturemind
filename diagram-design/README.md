## The 14 Diagram Types

| # | Type | Best For |
|---|---|---|
| 1 | Architecture | System topology, platform layers |
| 2 | Flowchart | Decision trees, KYC branching |
| 3 | Sequence | API auth, message timelines |
| 4 | State Machine | Entity lifecycle, status transitions |
| 5 | ER / Data Model | Schema, entity relationships |
| 6 | Timeline | Chronological journey, milestones |
| 7 | Swimlane | Cross-swarm handoff, compliance audit |
| 8 | Quadrant | Opportunity matrix, risk/effort |
| 9 | Nested | Hierarchy, containment |
| 10 | Tree | Parent→child, skill taxonomy |
| 11 | Layers | Technology stack, security model |
| 12 | Venn | Requirement overlap, intersection |
| 13 | Pyramid | Ranked priority, Maslow hierarchy |
| 14 | Consultant 2×2 | Strategic scenario matrix, named cells |

## VentureMind Diagrams in Use

**Architecture**: VentureMind system topology, agent communication graph
**Flowchart**: KYC tier decision tree, compliance escalation, onboarding
**Sequence**: Sovereign account creation, OAuth handshake, Vault handshake
**State Machine**: Entity lifecycle (DORMANT→ACTIVE→DISSOLVED), subscription tier transitions
**ER**: Core data model — Founder, Entity, SwarmSession, ExecutionLog
**Timeline**: 90-day founder journey, compliance milestone tracker
**Swimlane**: Multi-swarm sovereign execution, compliance audit, KYC escalation
**Quadrant**: Jurisdiction opportunity matrix (tax optimisation × infrastructure quality)
**Nested**: Nomad Flow platform hierarchy, agent taxonomy, 10-swarms hierarchy
**Tree**: Agent hierarchy (Central Lead → Strategy/Operator/Talent → sub-agents)
**Layers**: VentureMind security stack (UI → Agent Runtime → Orchestrator → Secure Layer → Infra)
**Venn**: Founder requirements overlap (Legal + Tax + Banking = Sovereign Sweet Spot)
**Pyramid**: Maslow hierarchy of founder needs (Info → Network → Mobility → Banking → Tax → Legal → Execution → Sovereignty)
**Consultant 2×2**: Jurisdiction strategy matrix (Validate / Prioritise / Defer / Avoid)
